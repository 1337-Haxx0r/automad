sys_cache_info: O armazenamento em cache é essencial para acelerar o site e geralmente deve estar ativado. No entanto, em algumas circunstâncias, convém desativar ou limpar seu cache.

-

sys_cache_clear: Limpar cache

-

sys_cache_clear_info: A limpeza do cache do site força uma nova verificação de páginas e arquivos atualizados. Esta ação não removerá nenhum arquivo de página ou imagem do diretório de cache.

-

sys_cache_monitor: O intervalo de verificação é

-

sys_cache_monitor_info: Quando o armazenamento em cache está ativado, o Automad verifica regularmente se há alterações no sistema de arquivos. A frequência dessas verificações pode ser definida aqui.

-

sys_cache_lifetime: O tempo de vida do cache é

-

sys_cache_lifetime_info: Todos os dados e páginas do Automad em cache têm uma vida útil máxima. Após a vida útil de um objeto exceder, seu cache será reconstruído quando solicitado.

-

sys_cache_purge: Diretório de Cache de Limpeza

-

sys_cache_purge_info: Ao contrário de simplesmente limpar o cache, a limpeza do diretório de cache também moverá todos os arquivos temporários, páginas excluídas, backups de atualização e imagens em cache para o diretório temporário do sistema. Isso forçará uma reconstrução completa do cache de imagens e páginas, além de uma limpeza de todas as páginas excluídas. Na maioria dos sistemas, o diretório temporário será limpo automaticamente de tempos em tempos ou na reinicialização. No entanto, em alguns sistemas, esse diretório deve ser limpo manualmente.

-

sys_debug_info: Quando a depuração estiver ativada, todos os processos do Automad serão registrados no console do seu navegador. A depuração é necessária apenas para desenvolvimento ou solução de problemas e deve ser desativada em todos os outros casos.

-

sys_headless_edit_info: A estrutura dos dados JSON retornados é controlada por um modelo. Você pode editar esse modelo usando a linguagem de modelo interna do Automad para personalizar campos usando variáveis, objetos e estruturas de controle.

-

sys_headless_info: A ativação do modo decapitado transforma o Automad em um sistema de gerenciamento de conteúdo focado inteiramente na criação de conteúdo sem uma camada de apresentação. Embora o Painel ainda possa ser usado para gerenciar conteúdo, todos os dados e arquivos são acessíveis por meio de uma API JSON somente leitura.

sys_cache_info: La memorizzazione nella cache è essenziale per velocizzare il tuo sito e generalmente dovrebbe essere abilitata. Tuttavia, in alcune circostanze potresti voler disabilitare o cancellare la cache.

-

sys_cache_clear: Cancella cache

-

sys_cache_clear_info: Svuotare la cache del sito impone una nuova scansione per pagine e file aggiornati. Questa azione non rimuoverà alcun file di pagina o immagine dalla directory della cache.

-

sys_cache_monitor: L'intervallo di controllo è

-

sys_cache_monitor_info: Quando la memorizzazione nella cache è abilitata, Automad verificherà periodicamente la presenza di modifiche nel file system. La frequenza di tali controlli può essere definita qui.

-

sys_cache_lifetime: La durata della cache è

-

sys_cache_lifetime_info: Tutti i dati e le pagine di Automad memorizzati nella cache hanno una durata massima. Al termine della durata di un oggetto, la sua cache verrà ricostruita quando richiesto.

-

sys_cache_purge: Elimina directory cache

-

sys_cache_purge_info: Contrariamente alla semplice pulizia della cache, l'eliminazione della directory della cache sposta anche tutti i file temporanei, le pagine eliminate, i backup degli aggiornamenti e le immagini memorizzate nella cache nella directory temprary del sistema. Ciò imporrà una ricostruzione completa dell'immagine e della cache della pagina, nonché una pulizia di tutte le pagine eliminate. Sulla maggior parte dei sistemi, la directory temporanea verrà ripulita automaticamente di volta in volta o al riavvio. Tuttavia, su alcuni sistemi tale directory deve essere cancellata manualmente.

-

sys_debug_info: Quando il debug è abilitato, tutti i processi di Automad verranno registrati sulla console del browser. Il debug è necessario solo per lo sviluppo o la risoluzione dei problemi e deve essere disabilitato in tutti gli altri casi.

-

sys_headless_edit_info: La struttura dei dati JSON restituiti è controllata da un modello. È possibile modificare quel modello utilizzando il linguaggio dei modelli integrato di Automad per personalizzare i campi utilizzando variabili, oggetti e strutture di controllo.

-

sys_headless_info: L'attivazione della modalità senza testa trasforma Automad in un sistema di gestione dei contenuti incentrato interamente sulla creazione di contenuti senza un livello di presentazione. Mentre Dashboard può ancora essere utilizzato per gestire il contenuto, tutti i dati e i file sono accessibili tramite un'API JSON di sola lettura.
